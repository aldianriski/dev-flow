# dev-flow Examples

Worked examples of projects bootstrapped with dev-flow.

## `node-express/`

Minimal Express service created via `node bin/dev-flow-init.js`. Shows project-specific files only — the `.claude/` tree and `docs/blueprint/` are generated by `dev-flow-init.js` at bootstrap time and are not checked in here (ADR-004).

**Contents**:
- `docs/` — rendered doc templates (AI_CONTEXT, ARCHITECTURE, DECISIONS, CHANGELOG, SETUP)
- `TODO.md` — sprint tracker, pre-filled for node-express stack
- `src/index.js` — minimal Express server
- `package.json` — project manifest

**To see the full bootstrapped `.claude/` tree**, run:
```bash
node bin/dev-flow-init.js
```
Then answer prompts: target directory, project name, stack (`node-express`), owner role.

## How to replicate this pattern

1. Clone dev-flow or add it as a local path.
2. Run: `node bin/dev-flow-init.js`
3. Answer prompts: target directory, project name, stack (`node-express`), owner role.
4. Add your application code alongside the scaffolded governance files.
5. Open `.claude/CLAUDE.md` and customize every `[CUSTOMIZE]` section for your project.
6. Run `/dev-flow` in Claude Code to start the workflow.
